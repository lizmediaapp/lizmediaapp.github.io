Local Files Help

Use LizMusic with music already stored on your Android device

LizMusic supports local music playback when your files already live on the device. If tracks do not appear, the issue is usually tied to storage access, folder location, or how the library was scanned.

Basic setup

  1. Store your music files on the device in a location Android allows the app to read.
  2. Grant the necessary media or storage access permission when LizMusic asks for it.
  3. Open the app and refresh or rescan the library if new files were just added.
  4. Check that your music files use formats LizMusic can recognize.

What local mode does not need

  • No WebDAV server is required.
  • No OneDrive account is required.
  • No internet connection is required for ordinary local playback once the files are on the device.

If your music is missing

Permission not granted

Open Android app settings and confirm LizMusic still has access to your media or storage.

Files were added recently

Refresh or rescan the library after copying new music onto the device.

Unsupported or unusual files

Verify the files are audio files rather than archives, shortcut files, or metadata-only entries.

Moved between apps or folders

If a file was relocated or deleted outside LizMusic, the old library entry may need a refresh before playback works again.

Need broader guidance?

If your local library works but cloud-connected playback does not, compare this page with the WebDAV and OneDrive help guides to isolate which storage path is failing.
